This document explains the smart capabilities incorporated into the project and why each was necessary.

Sample Transcript Generation and Automation

The OpenAI integration can produce narrative meeting transcripts that mirror the Fireflies format. This sample data accelerates testing and allows the automation pipeline to run even without real transcripts.

  • generateRandomSampleTranscript() returns cleaned narrative text for client testing.
  • The full response endpoint attaches execution metadata so developers can verify latency.

Real HubSpot Task Creation with Smart Priorities

Action items move to HubSpot as deals with normalized priorities, deadlines, and assignees. The normalization uses keyword detection to map urgency cues to HIGH/MEDIUM/LOW.

  • Priority defaults to LOW if the extractor cannot determine urgency, ensuring HubSpot receives valid enum values.
  • Reports capture raw HubSpot responses so you can trace which action item created which deal.

Resetting HubSpot State

The HubSpotDeleteDealsMain tool and deleteAllHubSpotDeals() flow remove all deals by pagination, allowing you to reset test environments quickly.

Smart Cleanup

  • The delete flow gathers IDs from paginated responses before issuing deletes.
  • The deletion report lists per-deal status so errant deletions can be audited.

Observability for Smart Flows

Metadata (timestamps/duration) and DTO reports help you detect when OpenAI takes longer or when HubSpot failures happen.
